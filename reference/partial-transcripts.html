Partial Transcripts & Live Cues

In voice, text deltas are the visual stream — the equivalent of token streaming on the research chat. Two channels, two timings.

Two transcript streams

WhoEventWhen it arrivesUI role
User conversation.item.input_audio_transcription.delta
…completed		 After VAD commits the buffer (post–speech-stop), not while the user is still talking "What we heard" confirmation — fills the thinking gap with legible text
Assistant response.output_audio_transcript.delta
…done		 While model audio plays (speaking) Visual stream — same job as streaming tokens in text chat

Session config (required for user text)

User transcription is off unless you enable it at session mint. Add under audio.input:

transcription: {
  model: 'gpt-4o-mini-transcribe',
  language: 'en',  // optional hint
}

Without this, you only get assistant transcript deltas — user turns stay invisible.

Reducer sketch

Keep transcript state separate from phase — same split as derivePhase vs message parts on text chat.

type Turn = { role: 'user' | 'assistant'; text: string };

type TranscriptState = {
  history: Turn[];
  draftUser: string;
  draftAssistant: string;
};

// delta → append to draft
// completed / done → push draft to history, clear draft
// response.cancelled → push partial assistant draft, then clear

UI treatments

PhaseTranscript surface
listening Phase cue only (mic pulse). User deltas haven't started — don't fake live captions.
thinking User draft may stream in — "Transcribing…" then partial text. This is your thinking-gap filler.
speaking Assistant draft streams word-by-word under the phase badge.
interrupted Keep partial assistant line in history (muted style). Not an error — same as text stopped.

Audio hygiene (from LR-0012)

  • Client: getUserMedia({ audio: { echoCancellation: true, noiseSuppression: true, autoGainControl: true } })
  • Server: audio.input.noise_reduction: { type: 'far_field' } on session mint

Choppy or self-interrupting VAD often traces to echo feeding back into the mic — fix audio before tuning thresholds.
