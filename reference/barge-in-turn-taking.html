Barge-in & Turn-taking

Voice inherits the interrupted-state design rules from text — preserve partial output, mark it as user-driven, recover by continuing the conversation. Two interrupt paths; one UX treatment.

Text → voice (same rules)

RuleText (stopped)Voice (interrupted)
Preserve Keep partial brief / streamed tokens Keep partial assistant transcript; server truncates unplayed audio on WebRTC
Mark "Stopped" badge — not an error "Interrupted" badge on the partial turn — not an error
Recover Regenerate in place Keep talking — VAD hands the floor back; no reconnect needed

Two ways to interrupt

PathTriggerServer eventWhen to offer
VAD barge-in User speaks while model is playing input_audio_buffer.speech_started → response.cancelled Default — natural conversation. Your job is to show it, not implement detection.
Explicit interrupt User taps Stop / Interrupt Client sends response.cancel → response.cancelled Noisy environments, accessibility, or when users won't talk over audio

Phase transitions

speaking
  ├─ response.cancelled ──► interrupted
  └─ output_audio_buffer.stopped ──► idle

interrupted
  └─ input_audio_buffer.speech_started ──► listening   // user takes the floor

listening / thinking / speaking …

interrupted is a brief beat — not a dead end. Phase badge copy should invite the next utterance ("Keep talking"), not sound like a failure.

Transcript flag

Extend turns pushed on response.cancelled with interrupted: true. Completed turns (…done) stay interrupted: false. Render interrupted lines muted + badge — same visual language as text stopped.

Send response.cancel (WebRTC)

dc.send(JSON.stringify({ type: 'response.cancel' }));

Only while speaking. On WebRTC the server manages output audio truncation; you don't need conversation.item.truncate client-side.

Session knobs (already on your probe)

  • turn_detection.type: 'server_vad' — automatic barge-in enabled
  • silence_duration_ms — how long silence before "thinking" (700 ms on your probe)
  • turn_detection.interrupt_response: false — disable auto-cancel (HITL / moderation flows only)
