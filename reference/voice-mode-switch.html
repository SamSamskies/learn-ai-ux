Voice Mode Switch

A polished Realtime probe is not a product. Productization means one brand, two modes — and an explicit choice about what voice is for vs what stays text-native.

Architecture fork (pick once)

PathBest forCost
Speech-to-speech (Realtime) Natural talk, barge-in, low first-audio latency Harder to reuse text HITL, citations, structured briefs mid-turn
Chained (STT → text agent → TTS) Extending an existing text agent; edit/approve before speak Higher latency; more moving parts

For a research assistant with citations and confirm dialogs, keep deep research on the text path. Use Realtime as a sibling mode for exploratory Q&A — not a drop-in replacement for useChat.

Mode-switch checklist

  1. Shared chrome — same product name, same header language on / and /voice.
  2. Explicit mode — Research | Voice (or mic entry) so users know which contract they're in.
  3. Voice instructions — short spoken answers; defer long lists / sources to text mode.
  4. Guard destructive exits — confirm before route-local work disappears.
  5. Carry intent — voice transcript → editable Research draft; never auto-send.
  6. Don't merge pipelines in v1 — two transports, one product.

Handoff boundary

DestroyCarryReview
Peer connection, microphone, Realtime phase, incomplete assistant audio Recent completed turns and a pending user utterance Seed the Research composer so the user can edit, refine, send, or discard

Use a one-shot, same-tab buffer such as sessionStorage when sibling routes do not share React state. Keep conversation text out of query parameters and remove the buffer after the destination consumes it.

Voice vs screen

Voice for the headline; screen for the data. Lists, URLs, citations, and comparisons fail in audio-only memory. Instruct the voice agent to summarize and point users to Research mode for depth.

Session instructions (pattern)

You are the voice mode of a research assistant.
Keep answers to 1–3 short sentences.
If the user needs sources, lists, or a written brief,
say so and suggest switching to Research mode.
Never invent citations aloud.
