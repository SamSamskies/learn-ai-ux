Inline Citations

Claim-level links from synthesis back to sources — so users can verify this sentence, not hunt through a footer.

Two layers of trust

LayerAnswersExample in your app
Scan height How grounded is this answer overall? TrustBar — confidence, provenance, source count
Claim level Which source backs this fact? Superscript [1] on key points → numbered source

Footer-only citations train users to trust unconditionally. Inline markers invite spot-checking at the moment of doubt.

Numbered markers pattern

Perplexity popularized this: superscript indices inline, full metadata in a numbered list below. Tradeoffs:

ProsScannable, compact, bidirectional (marker ↔ source), works with multiple sources per claim
ConsRequires model discipline on numbering; broken indices must degrade gracefully

Implementation checklist

Number sourcesid="source-1", source-2… on each item in the list; scroll-mt-* for anchor jumps
Render markersTransform [1] in text → superscript link to #source-1
Prompt + schemaTell the model to append [n] on cited key points; indices match numbered list order
Graceful fallbackOut-of-range [99] → render as plain text, not a dead link
Keep footerInline markers complement the full list — don't remove #brief-sources

Common mistakes

Markers without a numbered list[1] with nowhere to jump is decoration.
Full URLs inlineBreaks reading flow; save URLs for the source list.
Silent broken refsDead links erode trust faster than no citation.
Citations only on summaryKey points are where users spot-check facts — cite there first.
