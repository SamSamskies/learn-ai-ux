Voice Session Tools

Realtime function calling in a speech-to-speech session — register tools, detect function_call items, execute, return output, resume the spoken response.

Event loop (client)

  1. Register tools on the session (mint route or session.update).
  2. User speaks; VAD commits; model may emit response.done with output[].type === "function_call" instead of audio.
  3. Client runs your code (fetch, stash, navigate).
  4. Send conversation.item.create with type: "function_call_output" + matching call_id.
  5. Send response.create — model speaks using the tool result.

Register a tool (session mint)

tools: [
  {
    type: "function",
    name: "lookup_definition",
    description: "Short definition when the user asks what a term means.",
    parameters: {
      type: "object",
      properties: {
        term: { type: "string", description: "Word or phrase to define" },
      },
      required: ["term"],
    },
  },
],
tool_choice: "auto",

Detect a function call

// response.done — inspect output[], not transcript deltas
{
  type: "response.done",
  response: {
    output: [{
      type: "function_call",
      name: "lookup_definition",
      call_id: "call_abc",
      arguments: "{\"term\":\"RAG\"}",
    }],
  },
}

Return output + resume

// 1. Result item
{
  type: "conversation.item.create",
  item: {
    type: "function_call_output",
    call_id: "call_abc",
    output: JSON.stringify({ definition: "Retrieval-augmented generation…" }),
  },
}

// 2. Ask model to continue (spoken answer)
{ type: "response.create" }

UX patterns

GapWhy it hurtsFix
Silent tool run User hears nothing while fetch runs — feels like a freeze Label: "Looking up…" in dock or transcript; keep phase at thinking
Hidden side effects Tool pinned data or called an API — user can't audit Show a chip or system line: "Looked up: RAG"
Tool + barge-in User interrupts during fetch Abort in-flight fetch; ignore stale response.create
Research boundary Voice tool returns unsourced facts Stub/light lookup only; "Continue in Research" for citations

Symptom → fix

SymptomLikely causeFix
Model never calls the tool Weak description or tool not on session Sharpen description; confirm tools in mint response
Tool runs but no spoken answer Missing response.create after output Always send both output item and response.create
invalid_request_error on output call_id mismatch Use exact call_id from response.done
Double execution Handled same response.done twice Track processed call_ids or response ids
