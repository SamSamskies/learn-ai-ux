Text-Surface Retrofit Checklist

Eight lessons, one product surface. Use this to audit your research assistant — or any chat + structured-output app — before you call the text arc done.

Response lifecycle

CheckPass when…Lesson
Every state has a treatment idle, submitted, thinking, streaming, complete, stopped, and error each render distinct UI — not one spinner 0001
Stopped preserves partial output Abort keeps streamed text, labels it user-stopped, offers regenerate in place 0002
Transport errors are friendly useChat top-level error maps to plain copy — never raw error.message 0003
Thinking gap is legible Empty assistant message during connection shows a distinct thinking indicator, not a typing cursor 0004

Trust signals

CheckPass when…Lesson
Scan-height provenance TrustBar (or equivalent) shows tools used, source count, and calibrated confidence copy near the headline 0005
Claim-level citations Inline markers link key points to a numbered source list; post-processing covers model drift 0007

Human-in-the-loop

CheckPass when…Lesson
Destructive confirm New chat with history/uploads opens a modal naming what gets lost; empty state skips it 0006
Edit-before-send Refine lands in the composer with undo; Send stays manual; refine never auto-triggers a run 0008

Cross-cutting

Instant acknowledgementUser bubble appears and composer disables on Send before the server answers
Named progressResearch steps show labels ("Searching…") not generic "Working…"
Scroll on sendLatest turn stays in view — the wait doesn't feel like nothing happened
Strategic friction only where earnedConfirm dialogs and refine banners match real risk — not every action

Portfolio one-liner

When someone asks what you did: "I retrofitted a Phase 1 streaming demo into a product-grade text surface — deliberate lifecycle states, legible waits, calibrated trust signals, and human-in-the-loop at destructive, verification, and pre-run decision points."
