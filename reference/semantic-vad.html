Semantic VAD

When fixed silence can't tell "I'm thinking" from "I'm done," use a semantic classifier to chunk turns. Trade predictable latency for smarter boundaries.

Two VAD modes

ModeDetects end-of-turn by…Best for
server_vad Configured silence (silence_duration_ms) Snappy Q&A, commands, predictable latency budgets
semantic_vad Model scores whether the utterance sounds complete Reflective speech, mid-thought pauses, trailing "umm…"

Default is server_vad. Switch only when silence tuning clearly fails — not as a first dial.

semantic_vad config

turn_detection: {
  type: 'semantic_vad',
  eagerness: 'low',   // optional: low | medium | high | auto
  interrupt_response: true,
}

eagerness knob

ValueBehaviorWhen to use
high Chunk as soon as possible — closer to snappy server_vad Commands, rapid back-and-forth
medium / auto Default balance General conversation
low Wait longer; tolerate thinking pauses and trailing fillers Research Q&A, open-ended questions, storytelling

auto is equivalent to medium.

Symptom → dial

SymptomOn server_vadOn semantic_vad
Model jumps in at "The capital of France is…" Raise silence_duration_ms — but then all turns feel sluggish Switch to semantic_vad; try eagerness: 'low'
User trails off with "umm…" and gets cut off Hard to fix with silence alone semantic_vad extends timeout when completion score is low
Feels sluggish after short factual questions Lower silence_duration_ms Raise eagerness to medium or high
Latency readout varies turn-to-turn Expected — VAD tail is fixed Also expected — semantic wait is content-dependent; budget by feel, not one number

Product legibility

Surface the active mode in the voice UI ("Turn detection: semantic, low eagerness"). Users can't see VAD config — a label explains why some pauses feel longer and sets expectations after a mode change.

Decision rule

  1. Start with server_vad + measured silence_duration_ms.
  2. If raising silence fixes mid-thought cutoffs but breaks snappy Q&A, try semantic_vad.
  3. Tune eagerness before reverting — don't bounce between modes without a smoke test.
  4. Reconnect after session config changes — values are set at mint.
