Voice Arc Checklist

Eleven lessons on OpenAI Realtime. Use this to audit what you built — and to remember the product verdict: patterns ship; this voice stack does not (for your apps).

Product verdict (learner)

OpenAI Realtime speech-to-speech is not a surface you'd ship. STT is unreliable; image output via the voice path isn't usable. Keep voice as a lab probe. Research text remains the product.

What transfers (even if you delete /voice)

  • Phase reducers for nondeterministic waits — same discipline as text lifecycle.
  • Preserve + mark interrupted turns instead of discarding partial work.
  • Measure latency before tuning VAD / timeouts by feel.
  • Mode switch = state loss — confirm when tearing down local session state.
  • Handoff with edit-before-send when crossing modalities or precision boundaries.
  • Split-by-risk tools — auto-run cheap/safe; gate cost, side effects, creative prompts.
  • Don't bet UX on STT — confirm/edit on screen for anything that must be exact.

Audio hygiene (defaults if you ever reopen the probe)

Client: echoCancellation + noiseSuppression + autoGainControl. Server: noise_reduction: { type: 'far_field' }. Default VAD: server_vad, silence_duration_ms: 500.
