Edit-Before-Send

When AI rewrites or suggests a prompt, put the result in the composer — never auto-send. The user reviews, edits, and submits on their terms.

When to use

SituationWhy edit-before-send
Prompt enhancement AI clarifies a vague question — user must catch scope creep or wrong assumptions before a run
Suggested follow-ups Chips populate the composer; user tweaks wording before sending
Draft actions Agent proposes an email or query — user edits the payload before execute

Contrast with silent rewrite: many systems enhance prompts server-side without showing the user. That saves a click but hides intent drift and teaches nothing.

Prompt enhancer checklist

Never auto-sendRefined text lands in the textarea; Send stays manual
Label the changeBanner or chip: "Refined — review before sending"
Undo to originalOne click restores the user's draft if the rewrite misses
Preserve intentKeep nouns and constraints; don't invent facts or widen scope silently
Gate on inputMin character threshold or disabled state with clear affordance — not a mystery gray button
Disable while busyNo refine during an in-flight research stream

Common mistakes

Auto-send after refineUser loses agency; wrong rewrites burn compute and trust
No undoForces manual retyping when the enhancement is 80% right
Hidden enhancementServer rewrites without surfacing — user can't learn or correct
Refine on emptyProduces generic filler; needs the user's seed intent
