Realtime Voice Lifecycle

The states a speech-to-speech session moves through — and how they map to the text lifecycle you already shipped.

Text → voice mapping

Text (Phase 2)Voice (Phase 3)What's different
idle idle Session may stay open; mic is off or gated until user acts.
submitted connecting WebRTC handshake + ephemeral token mint — must be legible; silence feels broken.
thinking listening / thinking Two gaps: waiting for user speech end (VAD), then model reasoning before first audio.
streaming speaking Audio plays in real time; partial transcript deltas are the visual stream.
complete complete Turn done; mic re-opens for next utterance.
stopped interrupted Barge-in cancels model audio; preserve partial transcript, not an error.
error error Mic permission denied, connection drop, rate limit — plain copy + retry.

Realtime phases (derive from events)

PhaseTrigger (data channel)UI treatment
idle Session open, no active turn Connect / hold-to-talk affordance. Never leave the screen blank.
connecting Before session.created "Connecting…" — distinct from thinking. Disable double-connect.
listening input_audio_buffer.speech_started Live input indicator — user is being heard. Partial transcript optional.
thinking Speech stopped → before first response.output_audio.delta Named wait ("Thinking…") — the trust gap, same as text.
speaking response.output_audio.delta (or transcript deltas) Waveform or pulsing avatar + streaming transcript. Offer interrupt.
interrupted response.cancelled / barge-in Stop audio immediately; keep partial transcript; return to listening.
error Connection close, mic denied, API error event Friendly copy + reconnect. Never raw WebRTC errors.

Two architectures

PathWhenUX tradeoff
Speech-to-speech (Realtime API) Natural conversation, barge-in, low latency Less control over intermediate text; best feel, harder to debug.
Chained (STT → LLM → TTS) Approval flows, reuse text agent, durable transcripts Higher latency; each stage is a UI surface you own explicitly.

Latency budget (rule of thumb)

Conversational voice breaks around ~500–800 ms from end-of-speech to first model audio. Budget it:

  • Connection — warm session or show connecting state; don't cold-start on every utterance.
  • VAD tail — server turn detection adds silence before "thinking" starts.
  • First audio — the thinking gap; show visual progress if audio is late.
