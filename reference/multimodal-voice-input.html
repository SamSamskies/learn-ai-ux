Multimodal Voice Input

Realtime voice sessions accept discrete image attachments — not a live camera feed. Your app decides which images enter context and when.

What it is (and isn't)

Behavior
Is A user message with an input_image content part, sent over the data channel
Isn't Continuous video streaming, automatic screen capture, or vision baked into every audio frame

Images become conversation items the model can reference on the next spoken turn — ideal for charts, screenshots, and photos the user explicitly shares.

Wire format

{
  type: "conversation.item.create",
  item: {
    type: "message",
    role: "user",
    content: [
      {
        type: "input_image",
        image_url: "data:image/jpeg;base64,{bytes}",
        detail: "auto",   // optional: auto | low | high
      },
    ],
  },
}

Send via the WebRTC data channel (dataChannel.send(JSON.stringify(event))). With VAD enabled, attaching alone does not speak — the user asks aloud on the next turn.

Client constraints

  • Resize before encode. Full-resolution PNGs as base64 can choke the data channel. Cap the long edge (~1024 px) and prefer JPEG (~0.85 quality).
  • One attachment at a time for v1 — simpler UX and smaller payloads.
  • Show a thumbnail in the voice UI so users know the model can see what they attached.

UX patterns

PatternWhen
Attach → ask aloud Default. Image item lands in history; VAD commits the spoken question; model responds with audio.
Attach with caption text Add input_text in the same message when the image needs a label ("Q3 revenue slide").
Defer to Research When the answer needs citations or a written brief — voice describes what it sees; handoff carries transcript, not the image (v1 limitation).

Product legibility checklist

  1. Visible attach control (disabled until connected).
  2. Thumbnail or filename chip after attach — "Model can see this."
  3. Clear error if file is too large or wrong type.
  4. Session instructions mention screenshot/chart questions so spoken answers stay short.

Symptom → fix

SymptomLikely causeFix
Attach succeeds but model ignores image Image item not in context before response Attach before speaking; check data-channel send errors
Data channel silent / session drops on attach Payload too large Resize/compress client-side; cap file size (~4 MB raw)
Model hallucinates chart numbers Voice mode has no citation layer Short spoken summary + "Continue in Research" for sourced analysis
