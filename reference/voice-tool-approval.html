Voice Tool Approval

Gate side-effecting Realtime tools before execution — the model proposes, the user approves or rejects, then the session resumes with function_call_output.

When to auto-run vs gate

RiskVoice tool examplesClient behavior
Low — auto-run Definition lookup, read-only fetch Execute immediately; label the pause ("Looking up…")
Medium — approve once Stage Research draft, pin a note, send on user's behalf Hold function_call; show confirm UI with args; execute only on approve
High — strong confirm Delete data, purchase, share PII Modal with explicit consequence; default Cancel; consider blocking voice path entirely

Match friction to harm × scope × reversibility — same rule as text-surface HITL. Voice adds a constraint: the user may not be looking at the screen. Name the action in plain language and show the proposed args.

Approval loop (client)

  1. response.done with function_call for a gated tool.
  2. Parse name, arguments, call_id. Do not execute yet.
  3. Set dock label: "Waiting for your approval…" (or tool-specific copy).
  4. Show confirm UI — surface the parsed args (e.g. the research query).
  5. Approve: run side effect → send function_call_output with success payload → response.create.
  6. Reject: send function_call_output with { rejected: true } → response.create so the model can respond aloud.

Reject payload (model continues)

dc.send(JSON.stringify({
  type: "conversation.item.create",
  item: {
    type: "function_call_output",
    call_id: callId,
    output: JSON.stringify({ rejected: true, reason: "user_declined" }),
  },
}));
dc.send(JSON.stringify({ type: "response.create" }));

Without response.create, the session stalls after rejection. The model should acknowledge the decline in speech — tune instructions accordingly.

UX patterns

GapWhy it hurtsFix
Silent hold User hears nothing while approval UI waits Dock label + visible dialog; optional system line in transcript
Hidden args User approves blind Show the parsed query/topic in the dialog body
Stale approval User disconnects or barge-in during hold Clear pending state; auto-reject or drop stale response.create
Everything gated Prompt fatigue on harmless lookups Split tools: auto-run read-only, gate side effects only

Symptom → fix

SymptomLikely causeFix
Dialog never appears Tool routed to auto-run handler Branch on call.name before executing
Approve but model silent Missing response.create Send output item + response.create on both approve and reject
Model keeps re-proposing after reject Instructions too pushy "If rejected, acknowledge and offer an alternative — do not retry the tool."
Double staging Same call_id handled twice Track processed ids; clear pending on resolve
