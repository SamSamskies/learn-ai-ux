Voice → Research Handoff

Your confirm dialog makes mode-switch data loss honest. Today's win: “Continue in Research” ends the live voice session but carries its useful context into the Research composer as an editable draft.

Why this, why now

A warning protects users from accidental loss. A handoff helps them finish the job. The product should not preserve every implementation detail across modes; it should preserve intent.

  • Destroy: peer connection, microphone stream, Realtime phase, partial assistant audio.
  • Carry: completed transcript turns plus a pending user utterance.
  • Require review: seed the composer; never auto-send on arrival.
The core idea

A handoff is a lossy state boundary, not state synchronization. Translate the minimum useful context into the destination mode's native input.

Primary source — revisit this section (4 min) OpenAI · Voice agents — compare speech-to-speech with chained architecture. We are keeping the low-latency speech path and adding an explicit exit to the text path, not rebuilding a chain. Implementation companion: MDN · sessionStorage. Cheat sheet: voice mode switch.

Know this first (~4 min)

Why a one-shot buffer?

The routes are sibling pages, so page-local React state disappears on navigation. A small sessionStorage entry survives that navigation, stays scoped to the current tab, and can be deleted after one read.

Do not put the transcript in the URL: it leaks conversation text into history, is awkward to encode, and can grow beyond practical URL lengths.

The destination owns the final action

Voice produces context; Research accepts a draft. The user can edit, refine, or discard it before useChat sees anything. This reuses your edit-before-send HITL pattern.

Build it (~20 min)

  1. Create a one-shot handoff helper. Add lib/voice-handoff.ts. Keep only the last six turns so a casual voice session cannot silently become a giant research prompt. Include a pending user utterance, but exclude draftAssistant: incomplete model output is poor handoff context.

    const HANDOFF_KEY = "voiceResearchHandoff";

type VoiceTurn = {
  role: "user" | "assistant";
  text: string;
  interrupted?: boolean;
};

type VoiceHandoffSource = {
  history: VoiceTurn[];
  draftUser: string;
};

export function formatVoiceHandoff(
  source: VoiceHandoffSource,
): string | null {
  const pendingUser = source.draftUser.trim()
    ? [{ role: "user" as const, text: source.draftUser }]
    : [];
  const turns = [...source.history, ...pendingUser].slice(-6);
  if (turns.length === 0) return null;

  const transcript = turns.map((turn) => {
    const interrupted = turn.interrupted ? " [interrupted]" : "";
    return `${turn.role === "user" ? "User" : "Assistant"}${interrupted}: ${turn.text.trim()}`;
  });

  return [
    "Continue this as a source-backed research task.",
    "Use this voice transcript as context:",
    "",
    ...transcript,
  ].join("\n");
}

export function saveVoiceHandoff(draft: string) {
  sessionStorage.setItem(HANDOFF_KEY, draft);
}

export function takeVoiceHandoff(): string | null {
  const draft = sessionStorage.getItem(HANDOFF_KEY);
  sessionStorage.removeItem(HANDOFF_KEY);
  return draft;
}

    Optional durability check: add lib/voice-handoff.test.ts for empty input, pending user text, interruption labels, and the six-turn cap. Keep storage functions out of the unit test unless you configure a DOM environment.

  2. Translate voice state before navigating. In components/VoiceProbe/index.tsx, import the helper and add one exit function:

  if (draft) saveVoiceHandoff(draft);
  disconnect();
  router.push("/");
}

    Add a Continue in Research button when there is transcript text. Wire the existing switch confirmation to the same function, then change its copy:

    Because takeVoiceHandoff() deletes the key, refresh should not resurrect an already-consumed transcript. Verify this in development too, where React checks effect remount behavior.

Watch out

Do not promise that all page state survives. This handoff intentionally saves only voice context for the next task; it does not preserve the WebRTC connection or restore a previous Research timeline. Keep the UI copy precise.

Self-check

Why seed the composer instead of auto-sending?

Navigation should not trigger an expensive, nondeterministic action without review. An editable draft preserves user agency and composes with Refine and Undo.

Why carry a transcript instead of the Realtime session?

The destination needs intent, not transport state. Research has its own pipeline for durable text, sources, and citations.

What proves the buffer is truly one-shot?

Complete a handoff, then refresh Research. The draft should not reappear because the storage key was removed when consumed.
