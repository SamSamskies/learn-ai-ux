Partial Transcripts & Live Cues

Your phase badge tells users where they are. Partial transcripts tell them what is happening — the voice equivalent of streaming tokens. Today's win: a turn log on /voice with user confirmation text and a streaming assistant line.

Why this, why now

Lesson 0010 made every phase visible. But a badge alone still leaves two silent gaps:

  • Thinking gap — user finished talking, hears nothing. Text chat shows "Thinking…"; voice can show what was heard as transcription deltas arrive.
  • Speaking without sight — audio plays but users can't scan, quote, or verify. Assistant transcript deltas are your visual stream.

Same design move as Phase 2: make the nondeterministic wait legible. Different transport, same trust problem.

The core idea

Split phase (lifecycle reducer) from transcript (text accumulator). Phase drives the status chip; transcript drives the turn log — just like status vs message parts on useChat.

Primary source — read this (8 min) OpenAI · Realtime transcription — focus on "Handle transcript events" and the timing note about VAD commit. Cheat sheet: partial transcripts reference.

Know this first (~5 min)

User text lags; assistant text streams

StreamEventsTiming trap
User conversation.item.input_audio_transcription.delta → …completed Deltas start after VAD commits — usually during thinking, not listening. Don't fake live captions while they're still talking.
Assistant response.output_audio_transcript.delta → …done Arrives during speaking — true streaming, like tokens.

Enable user transcription at mint

User transcripts are off by default. Add transcription under audio.input in your session route — same place you already set noise_reduction and turn_detection.

Build it (~20 min)

Extend your existing VoiceProbe — no new route. Three files touched: session config, a transcript reducer, and the probe UI.

  1. Enable input transcription on the session. In app/api/realtime/session/route.ts, add under audio.input:

    transcription: {
  model: 'gpt-4o-mini-transcribe',
  language: 'en',
},

    Reconnect after this change — existing sessions won't pick it up.

  2. Add a transcript reducer. Create lib/realtime-transcript.ts — separate from reduceRealtimePhase:

    export type Turn = { role: 'user' | 'assistant'; text: string };

export type TranscriptState = {
  history: Turn[];
  draftUser: string;
  draftAssistant: string;
};

export const emptyTranscript: TranscriptState = {
  history: [], draftUser: '', draftAssistant: '',
};

export function reduceTranscript(
  state: TranscriptState,
  event: { type: string; delta?: string; transcript?: string },
): TranscriptState {
  switch (event.type) {
    case 'conversation.item.input_audio_transcription.delta':
      return { ...state, draftUser: state.draftUser + (event.delta ?? '') };
    case 'conversation.item.input_audio_transcription.completed': {
      const text = event.transcript ?? state.draftUser;
      if (!text.trim()) return { ...state, draftUser: '' };
      return {
        history: [...state.history, { role: 'user', text }],
        draftUser: '',
        draftAssistant: state.draftAssistant,
      };
    }
    case 'response.output_audio_transcript.delta':
    case 'response.audio_transcript.delta':
      return { ...state, draftAssistant: state.draftAssistant + (event.delta ?? '') };
    case 'response.output_audio_transcript.done':
    case 'response.audio_transcript.done': {
      const text = state.draftAssistant;
      if (!text.trim()) return state;
      return {
        history: [...state.history, { role: 'assistant', text }],
        draftUser: state.draftUser,
        draftAssistant: '',
      };
    }
    case 'response.cancelled': {
      if (!state.draftAssistant.trim()) return state;
      return {
        history: [...state.history, { role: 'assistant', text: state.draftAssistant }],
        draftUser: state.draftUser,
        draftAssistant: '',
      };
    }
    default:
      return state;
  }
}

  3. Wire both reducers on the data channel. In VoiceProbe, add transcript state and feed every event to both reducers:

    const [transcript, setTranscript] = useState(emptyTranscript);

dc.onmessage = (e) => {
  const event = JSON.parse(e.data);
  setPhase((p) => reduceRealtimePhase(p, event));
  setTranscript((t) => reduceTranscript(t, event));
};

    Clear transcript state on disconnect.

  4. Render a turn log under the phase badge. Simple layout — history first, then live drafts:

    {transcript.history.map((turn, i) => (
  <p key={i} className={turn.role === 'user' ? 'text-right' : ''}>
    <span className="text-xs uppercase text-on-surface-variant">{turn.role}</span>
    <br />{turn.text}
  </p>
))}
{transcript.draftUser && (
  <p className="text-right opacity-70">
    <span className="text-xs">you (transcribing…)</span><br />{transcript.draftUser}
  </p>
)}
{transcript.draftAssistant && (
  <p className="opacity-90">
    <span className="text-xs">assistant</span><br />{transcript.draftAssistant}
  </p>
)}

    Reuse your research-chat typography tokens. Muted style on interrupted partials is fine — same as a stopped text stream.

  5. Smoke-test the two streams. Connect, ask a short question, pause clearly (let VAD fire). Watch: user draft appears during thinking → finalizes → assistant draft streams during speaking. Barge-in mid-reply — partial assistant line should stay in history.

Watch out

No user text at all? Session mint is missing transcription. User text only after you stop talking? Expected — server VAD commits first. Choppy responses returning? Confirm echo cancellation is still on from lesson 0010.

Self-check

Why keep phase and transcript in separate reducers?

Same reason text chat splits SDK status from message content — lifecycle events and content deltas overlap but aren't 1:1. Merging them creates bugs when response.done fires before audio finishes.

Why not show user transcript while they're still speaking?

With server VAD, input transcription starts after buffer commit — not on every syllable. Showing fake live captions would lie about timing. Use the listening phase cue instead; user text fills the thinking gap.

What's the voice equivalent of streaming tokens?

response.output_audio_transcript.delta — append to a draft line while speaking. Users can read along with the audio.
