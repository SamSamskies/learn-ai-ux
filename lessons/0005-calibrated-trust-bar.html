Calibrated Trust Bar

The wait is legible now. The next win is trust — surfacing provenance, source count, and calibrated uncertainty at scan height so users know how hard to verify before they read a word of synthesis.

Where you actually are

Your BriefArticle already renders source pills and a confidence value. But they're buried in the footer, and confidence shows up as raw enum text:

Gap 1 — trust signals below the fold

Sources and provenance live after the summary and key points. Most users read the headline and first paragraph — then decide whether to trust. Citations they never see don't build trust.

Gap 2 — uncalibrated confidence

Confidence: {brief.confidence} renders high, medium, or low — internal telemetry, not a trust signal. Users can't act on it.

The principle

Trust signals help users judge whether to believe an output and how hard to verify it. Three rules from the Shape of AI citations pattern:

Place signals where people expect themScan height — under the headline, before the summary. Consistency beats polish.
Calibrate, don't pretendTranslate confidence into honest copy. A green check on every answer erodes trust faster than a caveat.
Make gaps explicitZero sources cited? Say so. Hiding missing provenance is worse than showing it.
Primary source (skim both) Shape of AI · Citations — where to place citations, metadata for scanning, and why missing sources must be explicit. Pair with Trust Builders for the broader pattern category.

Build it (~15 min)

  1. Create lib/confidence-copy.ts. One lookup table — enum in, calibrated copy + tone out:

    import type { ResearchBrief } from "@/lib/schemas";

export type ConfidenceLevel = ResearchBrief["confidence"];

export function confidenceCopy(level: ConfidenceLevel): {
  label: string;
  tone: "positive" | "caution" | "warning";
} {
  switch (level) {
    case "high":
      return {
        label: "Well-supported by retrieved sources",
        tone: "positive",
      };
    case "medium":
      return {
        label: "Partially supported — verify key claims",
        tone: "caution",
      };
    case "low":
      return {
        label: "Limited source coverage — treat as a draft",
        tone: "warning",
      };
  }
}

  2. Add TrustBar in components/ResearchChat/TrustBar.tsx — provenance chips, source count, and confidence badge in one scannable row:

    import { confidenceCopy } from "@/lib/confidence-copy";
import type { ResearchBrief } from "@/lib/schemas";
import type { Source } from "@/lib/research-state";
import { dedupeSources } from "@/lib/research-state";

const toneClass = {
  positive: "border-good/30 bg-good/5 text-good",
  caution: "border-warn/30 bg-warn/5 text-warn",
  warning: "border-accent/30 bg-accent-soft text-accent",
} as const;

export function TrustBar({
  brief,
  sources,
  searched,
  searchedDocs,
}: {
  brief: ResearchBrief;
  sources: Source[];
  searched: boolean;
  searchedDocs: boolean;
}) {
  const { label, tone } = confidenceCopy(brief.confidence);
  const count = dedupeSources(sources).length;

  return (
    <div
      className="mt-4 flex flex-wrap items-center gap-2"
      aria-label="Trust and provenance"
    >
      <span
        className={`rounded-lg border px-2.5 py-1 text-xs font-medium ${toneClass[tone]}`}
      >
        {label}
      </span>

      {searched && (
        <span className="rounded-lg border border-outline-variant px-2.5 py-1 text-xs text-on-surface-variant">
          Web search
        </span>
      )}
      {searchedDocs && (
        <span className="rounded-lg border border-outline-variant px-2.5 py-1 text-xs text-on-surface-variant">
          Your documents
        </span>
      )}
      {!searched && !searchedDocs && (
        <span className="rounded-lg border border-outline-variant px-2.5 py-1 text-xs text-on-surface-variant">
          Model knowledge only
        </span>
      )}

      {count > 0 ? (
        <a
          href="#brief-sources"
          className="rounded-lg border border-outline-variant px-2.5 py-1 text-xs text-primary hover:bg-surface-container-low"
        >
          {count} {count === 1 ? "source" : "sources"}
        </a>
      ) : (
        <span className="text-xs text-on-surface-variant">
          No sources cited
        </span>
      )}
    </div>
  );
}

    Map tone to your existing Tailwind tokens — or use neutral borders if you prefer a quieter bar. The copy matters more than the color.

  3. Wire it into BriefArticle — trust bar after the headline, anchor on the sources section, delete the footer dump:

// on the sources block:
<div id="brief-sources" className="mt-8 scroll-mt-4">

// remove the footer <div> with "Grounded with…" and "Confidence:"

  4. Test three scenarios. Ask questions that hit different trust profiles:

    • Web-grounded: a current-events question → web chip + source count + high/medium confidence
    • Doc-grounded: upload a PDF, ask about it → "Your documents" chip + file source pills
    • Thin coverage: an obscure niche question → low confidence copy + "No sources cited" if annotations are empty

    The trust bar should be visible without scrolling past the headline.

What you ship

Self-check

Why is Confidence: high worse than calibrated copy?

It exposes internal schema values users can't act on. Calibrated copy tells them what to do — verify key claims, treat as draft — which is the point of a trust signal.

When should you show "No sources cited" instead of hiding the sources row?

Whenever dedupeSources(sources) is empty after a completed brief. Explicit gaps maintain trust; users assume uncited answers are verified when you stay silent.
