AI Product UX · Phase 2 · Lesson 0006

Confirm Destructive Actions

Trust signals are at scan height now. The next win is human-in-the-loop — gating irreversible actions so speed never costs the user their work.

Where you actually are

Your header has a New Chat button that fires immediately:

Gap — one-click data loss

newChat() stops the stream, DELETEs the server session (vector store + uploaded docs), rotates sessionId, and clears client state. One mis-click after a long research thread — gone. No undo.

That's a destructive action: high harm (loss of work), wide scope (entire session), low reversibility (no recycle bin). It should use strategic friction — a deliberate pause that builds confidence instead of eroding it.

The principle

Not every AI interaction needs a confirmation. Match friction to harm × scope × reversibility:

Low riskSearch, regenerate, copy brief → no dialog; undo/recover is enough.
High riskDelete session, overwrite data, send on user's behalf → confirm before execute.
Nothing to loseEmpty chat, no uploads → skip the dialog; friction with no benefit is prompt fatigue.

A good confirm dialog names what gets lost in plain language — not "Are you sure?"

Primary source (skim) Shape of AI · Verification — when to require human verification, matching friction to risk, and keeping lightweight checks lightweight. Pair with The Trust-Latency Gap for why intentional slowdowns build trust on high-stakes actions.

Build it (~15 min)

  1. Create components/ResearchChat/ConfirmDialog.tsx. A reusable gate — accessible, focus-safe, backdrop dismiss:

    "use client";

import { useEffect, useId, useRef } from "react";

export function ConfirmDialog({
  open,
  title,
  description,
  confirmLabel = "Confirm",
  cancelLabel = "Cancel",
  destructive = false,
  onConfirm,
  onCancel,
}: {
  open: boolean;
  title: string;
  description: string;
  confirmLabel?: string;
  cancelLabel?: string;
  destructive?: boolean;
  onConfirm: () => void;
  onCancel: () => void;
}) {
  const titleId = useId();
  const cancelRef = useRef<HTMLButtonElement>(null);

  useEffect(() => {
    if (!open) return;
    cancelRef.current?.focus();
    const onKeyDown = (e: KeyboardEvent) => {
      if (e.key === "Escape") onCancel();
    };
    document.addEventListener("keydown", onKeyDown);
    return () => document.removeEventListener("keydown", onKeyDown);
  }, [open, onCancel]);

  if (!open) return null;

  return (
    <div
      className="fixed inset-0 z-50 flex items-center justify-center p-4"
      onClick={onCancel}
    >
      <div
        className="absolute inset-0 bg-inverse-surface/40"
        aria-hidden
      />
      <div
        role="dialog"
        aria-modal="true"
        aria-labelledby={titleId}
        className="relative z-10 w-full max-w-md rounded-xl border border-outline-variant bg-background p-6 shadow-lg"
        onClick={(e) => e.stopPropagation()}
      >
        <h2
          id={titleId}
          className="text-lg font-semibold text-foreground"
        >
          {title}
        </h2>
        <p className="mt-2 text-sm leading-relaxed text-on-surface-variant">
          {description}
        </p>
        <div className="mt-6 flex justify-end gap-2">
          <button
            ref={cancelRef}
            type="button"
            onClick={onCancel}
            className="rounded-lg border border-outline-variant px-4 py-2 text-sm font-medium text-foreground hover:bg-surface-container-low"
          >
            {cancelLabel}
          </button>
          <button
            type="button"
            onClick={onConfirm}
            className={
              destructive
                ? "rounded-lg bg-error px-4 py-2 text-sm font-medium text-on-error hover:bg-error/90"
                : "rounded-lg bg-primary px-4 py-2 text-sm font-medium text-on-primary hover:bg-primary-dark"
            }
          >
            {confirmLabel}
          </button>
        </div>
      </div>
    </div>
  );
}

    import { ConfirmDialog } from "./ConfirmDialog";

const [confirmNewChat, setConfirmNewChat] = useState(false);

function requestNewChat() {
  if (isEmpty && uploadedFiles.length === 0) {

// header button:

// before closing </div> of the root layout:
<ConfirmDialog
  open={confirmNewChat}
  title="Start a new chat?"
  description="This clears your conversation and uploaded documents from this session. This can't be undone."
  confirmLabel="New chat"
  destructive
  onConfirm={() => {
    setConfirmNewChat(false);
    void newChat();
  }}
  onCancel={() => setConfirmNewChat(false)}
/>

  3. Test three paths.

    • Empty state: fresh load → New Chat runs immediately, no dialog
    • With history: after one completed brief → dialog appears; Cancel keeps everything; confirm clears the thread
    • With uploads: attach a PDF (even before asking a question) → dialog mentions documents; confirm clears the file chips

    Also verify: dialog won't open while busy (button stays disabled), Escape and backdrop click dismiss without side effects.

Self-check

Why skip the dialog on an empty chat?

Confirmation friction should match risk. With no turns and no uploads, there's nothing to lose — a dialog would only train users to click through without reading.

Why focus Cancel instead of the destructive button?

Default-to-safe: the reversible action gets keyboard focus so a hurried Enter/Escape doesn't accidentally wipe a session.
