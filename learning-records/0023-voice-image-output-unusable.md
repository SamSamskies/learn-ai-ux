# Voice-triggered image gen quality is not shippable

After lesson 0020, generated illustrations were bad enough the learner would never use the feature. Editable prompt gating and cheap model settings don't fix output that fails the "would I show this to anyone?" bar.

**Implications:** Don't invest further in voice→Images API as a product surface on this stack. The UX pattern (confirm prompt on screen, show picture, speak a caption) still transfers if image quality improves elsewhere — the verdict is on quality, not on the gate.
